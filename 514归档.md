## 8. 教训

1. **subagent 自写自测 = 无效验证**：`title_required` bug 自测通过，用户一用就 crash。必须 subagent 隔离审查。
2. **静默 no-op 是最难 debug 的 bug**：`writePlan(null, ...)` 因为有 `if (!db) return;` 保护，整个 Scout 流程"跑完了"但啥也没写进去。根因是上游传错参数类型，没有任何报错。
3. **mutation 函数的副作用必须显式隔离**：`registry.setProjectGoal()` 看起来是"纯 setter"，实际内部 `saveRegistry()` 写磁盘。所有 smoke/test 必须先 sandbox HOME 再 require mutation 模块。
4. **panel 字段名漂移是系统性问题**：本次 session 发现 4+ 处 `goal.text` vs `goal.title`、`raised_at` vs `created_at` 类型的漂移。建议 IPC 层做 schema 验证。
5. **Mode A 成功运行但 UI stale 会造成"假死"误判**：CC 已经 commit `b377215` on agent-game-platform，但 panel 是 push 前起的，前端不认识新的 phase pill，导致用户以为 agent 空闲。
