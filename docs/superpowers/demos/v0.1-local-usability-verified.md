## Gaps not closed (out-of-scope for this verification)

- **Panel UI visual render** — boot logs prove the Electron main process is alive and the SQLite handle works; whether the panel HTML *displays correctly* needs eyes-on (or a CDP smoke against the packaged binary; the existing `cdp-client.mjs` targets dev-mode `npm start`, not the packaged `.exe`).
- **Mac side** — config exists for `.dmg`, no Mac available on this machine. Verification needs either a Mac or a GitHub Actions runner.
- **Real Claude Code session against the installed `cairn-wedge`** — installing rewrites `.mcp.json`, but verifying Claude Code actually picks it up requires a Claude Code restart in the target repo, which is a user action.

These three gaps don't affect the "本地可用" gate the user named — that gate is about cold-path install correctness, which is now fully evidenced.

## Reproducing

```bash
# pick any tmp dir
FRESH=$(mktemp -d -t cairn-fresh-XXXX)
cd "$FRESH"
git clone https://github.com/Upp-Ljl/Cairn.git
cd Cairn

# build the three packages
for pkg in daemon mcp-server desktop-shell; do
  (cd packages/$pkg && npm install --no-audit --no-fund && npm run build 2>/dev/null || true)
done

# verify CLI without touching anything
node packages/mcp-server/dist/cli/install.js --version  # → cairn 0.1.0
node packages/mcp-server/dist/cli/install.js --help     # usage

# pick any tmp target repo + initialize as git
TARGET=$(mktemp -d -t cairn-target-XXXX)
cd "$TARGET"
git init -b main && git commit --allow-empty -m "init"

# preview, then install
node "$FRESH/Cairn/packages/mcp-server/dist/cli/install.js" --dry-run
node "$FRESH/Cairn/packages/mcp-server/dist/cli/install.js"

# end-to-end on the live testbed (requires the clone at expected path
# or CAIRN_DOGFOOD_REPO_PATH=<your-clone> set)
node "$FRESH/Cairn/packages/desktop-shell/scripts/dogfood-managed-project-loop.mjs"
# expect: 21/21 PASS
```

The local-usability gate the user named is now met. The remaining v0.1 release-path decisions are:

Until those, distribution path for teammates is: `git clone` from the public repo + follow this verified cold path. The README onboarding section captures the user-facing version of these same steps.
