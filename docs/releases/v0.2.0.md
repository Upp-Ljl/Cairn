## What changed since v0.1.x

This release is the **cockpit redesign + Mode B** ship, closing all 17
product constraints from the 2026-05-14 CEO grill.

### Panel cockpit — new layout

Module order changed to match CEO 17-constraint spec:

1. **M1 State Strip** — Whole + Mentor-saved badge + Last-24h glance
2. **M2 Mentor + Todolist** — primary module, was hidden, now first-class
   - Status header: state · last nudge · today's decisions · 🛤 lane indicator
   - Todolist with three sources: 🤖 agent proposals · 🧑‍🏫 Mentor recs · 🐤 you
   - `[派给]` / `[Approve]` buttons → dispatch_requests (Cairn kernel R1-R6)
   - Add-input for your own todos
   - **NEW**: batch-select todos → "Authorize N as lane" (Mode B slice 5)
3. **M3 Steer** — single line to any session, with session-target dropdown
4. **M4 Sessions** — every agent attributed to this project (working / blocked / idle / stale). idle is first-class.
5. **M5 Safety / Rewind** — kept

Plus:
- **🛤 Mode B · Lanes** module between Todolist and Steer — Continuous Iteration chain status
- **L2 Session Timeline** — click any session card → see agent execution脉络 (subagent缩进 tree)
- **First-launch wizard** — non-developer onboarding (3 screens)

### Mode B Continuous Iteration (NEW)

The "走开就行" feature. User authorizes N candidate task_ids → Cairn
auto-watches them in sequence, stops at each REVIEWED state for
explicit user advance (PRODUCT.md §1.3 #4a never auto-accepts).

Slices shipped in v0.2.0:
1. Lane data layer + IPC (`cockpit-lane.cjs`)
2. cockpit-state.lanes + panel UI Module
3. mentor-tick detects WAITING_REVIEW → flips lane to REVIEW + Mentor nudge
4. "+ New lane" button + creation form
5. Batch-authorize from Todolist (checkbox per row + footer button)

### Mentor

- **Rule C off-goal drift** (LLM-judged) — domain-mismatch detection
- **Rule C strict-mode** — high-confidence verdicts nudge immediately (1 strike instead of 2)
- **Real-LLM dogfood** — verified against minimax provider, real-world prompt

### Session timeline (新 protocol)

`session_timeline/<agent_id>/<ulid>` scratchpad namespace. Agents write
events via existing `cairn.scratchpad.write` (no new MCP tool). **Plus**:

- **Kernel auto-instrumentation** — Cairn itself writes timeline events on every task state transition (CREATED → RUNNING → BLOCKED → ANSWERED → WAITING_REVIEW → DONE / CANCELLED). Agents who don't adopt the cairn-aware skill still get a usable timeline.

### Ship pipeline

- **GitHub Actions** `.github/workflows/release.yml` — win + mac matrix
- **electron-builder** `extraResources` bundles mcp-server into .exe / .dmg
  (no more "clone + npm install + cairn install" for non-developers)
- **build/icon.icns** — Mac dmg icon (3099 bytes, real .icns format)

### Workflow rules

`docs/workflow/SELF-REPORT-STOP.md` gained 3 new fields:

- **13** `mid_work_status_report` — don't surface progress mid-run
- **14** `push_block_misread_as_dev_block` — push fail ≠ dev stop
- **15** `subagent_running_misread_as_stop` — subagent in flight ≠ idle

## Download

- **Windows** (`Cairn Setup 0.2.0.exe`, NSIS installer, ~85MB) — works on Win10/11
- **macOS** (`Cairn-0.2.0.dmg`, x64 + arm64, ~90MB) — **unsigned**, Gatekeeper warning on first launch. Right-click → Open to bypass (or `xattr -d com.apple.quarantine /Applications/Cairn.app`)
- **Linux** — not yet (electron-builder linux target later)
